# 提交前检查清单

[返回索引](./README-zh.md)

在提交代码前，请确认以下所有项目：

---

## 文档与注释
- [ ] 没有创建不必要的总结文档
- [ ] 所有注释使用中文（专业术语除外）
- [ ] 代码中没有使用 Emoji
- [ ] 必须维护的文档已更新（README、CHANGELOG、端口说明）

---

## 代码质量
- [ ] 没有硬编码的状态码、类型码（使用枚举）
- [ ] 所有可配置值已提取到配置文件
- [ ] 使用 jakarta.* 包而不是 javax.*
- [ ] 领域模型的私有构造函数添加了 @JsonCreator
- [ ] Optional 只用于返回值，不用于参数或字段
- [ ] Stream 链式调用不超过 5 步
- [ ] Lombok @Data 正确处理了 equals/hashCode
- [ ] 代码格式通过 `mvn spotless:check`
- [ ] 单元测试覆盖率达标（核心模块 70%+）

---

## Git 提交
- [ ] Commit message 符合 Conventional Commits 规范
- [ ] 一次提交只做一件事
- [ ] 没有使用 update、test、tmp 等无意义提交信息

---

## 安全
- [ ] 没有提交敏感信息（密码、密钥、Token）
- [ ] 敏感配置支持环境变量覆盖
- [ ] 日志中的敏感字段已脱敏
- [ ] 生产环境配置文件已加入 .gitignore

---

## 基础设施
- [ ] 使用 docker-compose 管理容器
- [ ] 端口配置正确，没有冲突（参考项目的 infrastructure-ports.md）
- [ ] Redis Key 命名符合规范
- [ ] 容器资源限制已配置（CPU/Memory）

---

## 日志与监控
- [ ] 日志包含 traceId、userId 等关键信息
- [ ] 没有打印大对象或在循环中打印日志
- [ ] 错误日志包含完整堆栈信息
- [ ] 关键业务指标已添加监控

---

## 接口设计
- [ ] RESTful API 命名规范
- [ ] 接口版本控制正确（/api/v1/...）
- [ ] 幂等性设计（支付、订单等场景）
- [ ] 错误码定义清晰且分段合理
- [ ] 分页接口限制最大页面大小
- [ ] 排序字段使用白名单校验

---

## 数据库
- [ ] 表包含必须字段（created_at、updated_at、deleted）
- [ ] 索引有明确的用途说明
- [ ] 没有使用 SELECT *
- [ ] 没有无 WHERE 条件的 UPDATE/DELETE
- [ ] 事务范围合理（不包含外部 API 调用）

---

## 测试
- [ ] PowerShell 脚本使用 Invoke-WebRequest
- [ ] 测试用例覆盖全面（正常、异常、边界、安全）
- [ ] 测试数据使用时间戳隔离
- [ ] 集成测试使用 Testcontainers（如适用）
- [ ] 单元测试命名符合 should_xxx_when_yyy 格式
- [ ] 使用 AssertJ 进行断言

---

## 缓存与并发
- [ ] 缓存过期时间已配置（避免永不过期）
- [ ] 缓存穿透防护已实现（空值缓存或布隆过滤器）
- [ ] 缓存更新策略正确（删除而不是更新）
- [ ] 分布式锁设置了超时时间
- [ ] 锁粒度合理（不要使用全局锁）
- [ ] 幂等性设计已实现（Token/唯一索引/状态机）

---

## PR 审查
- [ ] PR 包含背景说明和改动点
- [ ] PR 包含影响范围和回滚方案
- [ ] 单元测试通过
- [ ] 静态检查通过
- [ ] 至少 1 人代码审查通过
