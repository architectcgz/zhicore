# 评论数据生成说明

本文档说明如何生成测试评论数据。

## 概述

评论数据生成脚本 (`Generate-Comments.ps1`) 为已发布文章生成评论数据，包括：

- 一级评论（顶级评论）
- 嵌套回复（二级评论）
- 评论点赞记录
- 评论统计数据（comment_stats）
- 文章评论计数更新（post_stats.comment_count）

## 前置条件

1. **服务运行**：
   - blog-id-generator 服务（端口 8088）
   - blog-gateway 服务（端口 8000）
   - blog-comment 服务（端口 8083）

2. **数据准备**：
   - 已执行用户数据生成脚本（`Execute-UserGeneration.ps1`）
   - 已执行文章数据生成脚本（`Generate-Posts.ps1`）
   - 至少有一些已发布状态的文章

## 使用方法

### 基本用法

```powershell
# 使用默认配置
.\Generate-Comments.ps1

# 预览模式（不实际创建数据）
.\Generate-Comments.ps1 -DryRun

# 使用自定义配置文件
.\Generate-Comments.ps1 -ConfigPath ".\custom-config.json"
```

### 自定义参数

```powershell
# 自定义每篇文章的评论数量
.\Generate-Comments.ps1 -MinCommentsPerPost 10 -MaxCommentsPerPost 50

# 自定义回复比例（30% 的评论会收到回复）
.\Generate-Comments.ps1 -ReplyRatio 0.3

# 组合使用
.\Generate-Comments.ps1 -MinCommentsPerPost 5 -MaxCommentsPerPost 30 -ReplyRatio 0.4
```

## 配置说明

配置文件 (`test-data-config.json`) 中的相关配置：

```json
{
  "comments": {
    "perPost": {
      "min": 5,
      "max": 30
    },
    "replyRatio": 0.3
  }
}
```

### 配置项说明

- `perPost.min`: 每篇文章最少评论数（默认：5）
- `perPost.max`: 每篇文章最多评论数（默认：30）
- `replyRatio`: 回复比例，0-1 之间（默认：0.3，即 30% 的评论会收到回复）

## 生成流程

### 1. 验证服务可用性

脚本首先验证必需的服务是否正常运行：

- ID Generator 服务
- API Gateway 服务

### 2. 获取已发布文章

从 API 获取所有已发布状态的文章列表。

### 3. 获取测试用户

从 API 获取所有测试用户列表。

### 4. 生成一级评论

为每篇已发布文章生成 5-30 条一级评论：

- 随机选择评论作者
- 使用预定义的评论内容模板
- 调用 API 创建评论
- 自动创建 comment_stats 记录
- 自动更新 post_stats.comment_count

### 5. 生成嵌套回复

为约 30% 的一级评论生成回复：

- 每条被回复的评论生成 1-5 条回复
- 正确设置 root_id、parent_id、reply_to_user_id
- 自动更新 comment_stats.reply_count

### 6. 生成评论点赞

为约 30% 的评论生成点赞：

- 每条评论生成 1-20 个点赞
- 随机选择点赞用户
- 确保用户不重复点赞
- 自动更新 comment_stats.like_count

## 数据结构

### 一级评论

```json
{
  "postId": 1234567890,
  "content": "写得很好，学到了很多！"
}
```

### 回复评论

```json
{
  "postId": 1234567890,
  "content": "同意你的观点！",
  "rootId": 9876543210,
  "replyToCommentId": 9876543210
}
```

### 评论点赞

通过 API 调用实现：

```
POST /api/v1/comments/{commentId}/like
Headers:
  X-App-Id: test-app
  X-User-Id: {userId}
```

- ✅ **Requirement 6.3**: 为至少 30% 的评论生成 1-5 条回复

- ✅ **Requirement 6.5**: 确保 parent_id 指向有效的评论
- ✅ **Requirement 6.6**: 为每条评论创建 comment_stats 记录
- ✅ **Requirement 6.7**: 为部分评论生成点赞记录

- ✅ **Requirement 6.9**: 更新 post_stats 表的 comment_count

## 故障排查

### 问题 1: 未找到已发布文章

**错误信息**：
```
✗ 获取已发布文章失败: 未找到已发布文章
```

**解决方案**：
1. 确认已执行文章数据生成脚本
2. 确认至少有一些文章处于已发布状态
3. 检查 blog-post 服务是否正常运行

### 问题 2: 未找到测试用户

**错误信息**：
```
✗ 获取测试用户失败: 未找到测试用户
```

**解决方案**：
1. 确认已执行用户数据生成脚本
2. 检查 blog-user 服务是否正常运行

### 问题 3: 创建评论失败

**错误信息**：
```
⚠ 创建评论失败: API 调用失败
```

**解决方案**：
1. 检查 blog-comment 服务是否正常运行
2. 检查网络连接
3. 查看服务日志获取详细错误信息
4. 确认 API 地址配置正确

### 问题 4: ID Generator 服务不可用

**错误信息**：
```
✗ ID Generator 服务不可用
```

**解决方案**：
1. 启动 blog-id-generator 服务
2. 确认服务端口为 8088
3. 检查服务健康状态：`curl http://localhost:8088/actuator/health`

## 注意事项

1. **执行顺序**：必须先执行用户和文章数据生成脚本
2. **服务依赖**：确保所有必需的服务都已启动
3. **数据量**：评论数据量较大，生成过程可能需要几分钟
4. **重复执行**：可以重复执行脚本，会为现有文章添加更多评论
5. **API 限流**：如果遇到限流，脚本会自动重试
