# post_stats - 文章统计表

## 表信息

| 属性 | 值 |
|------|-----|
| 表名 | post_stats |
| 描述 | 文章统计信息表 |
| 主键 | PostId (long) |

## 字段定义

| 字段名 | 类型 | 可空 | 默认值 | 说明 |
|--------|------|------|--------|------|
| PostId | long | 否 | - | 文章ID（主键，关联posts表） |
| ViewCount | long | 否 | 0 | 文章阅读数 |
| LikeCount | int | 否 | 0 | 文章点赞数量 |
| CommentCount | int | 否 | 0 | 文章评论数量 |
| FavoriteCount | int | 否 | 0 | 文章收藏数量 |
| UpdateTime | DateTimeOffset | 否 | UtcNow | 统计信息更新时间 |

## 索引

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| PK_post_stats | PostId | 主键 | 主键索引 |

## 关联关系

| 关联表 | 关联字段 | 关系类型 | 说明 |
|--------|----------|----------|------|
| posts | PostId | 一对一 | 关联文章 |

## 设计说明

将统计数据分离到独立表的原因：
1. **减少锁竞争**: 统计更新不会锁定文章主表
2. **优化写入**: 高频更新的统计数据独立存储
3. **便于缓存**: 统计数据可以独立缓存和失效
4. **热度计算**: 便于基于统计数据计算文章热度
