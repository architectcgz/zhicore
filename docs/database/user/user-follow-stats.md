# user_follow_stats - 用户关注统计表

## 表信息

| 属性 | 值 |
|------|-----|
| 表名 | user_follow_stats |
| 描述 | 用户关注/粉丝数量统计表 |
| 主键 | UserId (string) |

## 字段定义

| 字段名 | 类型 | 可空 | 默认值 | 说明 |
|--------|------|------|--------|------|
| UserId | string | 否 | - | 用户ID（主键） |
| FollowingCount | int | 否 | 0 | 关注数量（我关注了多少人） |
| FollowerCount | int | 否 | 0 | 粉丝数量（多少人关注了我） |
| UpdateTime | DateTimeOffset | 否 | - | 统计更新时间 |

## 索引

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| PK_user_follow_stats | UserId | 主键 | 主键索引 |

## 关联关系

| 关联表 | 关联字段 | 关系类型 | 说明 |
|--------|----------|----------|------|
| AspNetUsers | UserId | 一对一 | 关联用户 |

## 设计说明

将统计数据分离到独立表的原因：
1. 减少主表的写入压力
2. 统计数据更新频繁，分离后可以独立优化
3. 便于缓存策略的实现
