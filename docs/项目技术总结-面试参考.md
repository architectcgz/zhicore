## 🎯 项目概述

这是一个采用前后端分离架构的全栈博客系统，具备企业级特性和高性能设计。项目突出了大型Web应用的技术难点解决方案，是展示全栈开发能力和系统优化能力的优秀案例。

## 🎯 核心技术亮点

### 1. 性能优化专项 ⭐⭐⭐

**问题背景**：核心接口响应时间高达1.45秒，严重影响用户体验

**优化成果**：
- 用户文章列表：1.45秒 → 572毫秒（**60.6%提升**）
- 话题搜索：1秒+ → 200-400毫秒（**60-80%提升**）
- 用户关注列表：800毫秒+ → 150-300毫秒（**62-81%提升**）

**技术创新点**：

#### ① 雪花算法ID优化游标分页
```csharp
// 创新：直接使用雪花ID作为游标，无需复合键
public class CursorInfo
{
    public long Id { get; set; }  // 雪花ID天然包含时间戳
}
// 简单的ID比较替代复杂的时间+ID比较
query = query.Where(e => e.Id < cursorInfo.Id);
```

#### ② 批量缓存调用优化
```csharp
// 优化前：20次串行Redis调用
// 优化后：2次批量调用，网络往返减少90%
var keys = postIds.Select(id => $"post:like:count:{id}").ToArray();
var values = await _redis.StringGetAsync(keys);
```

#### ③ N+1查询根治
- 使用EF Core的Include预加载机制
- 数据库查询从21次降至3次
- 消除了典型的ORM性能陷阱

## 🛠️ 技术难点与解决方案

### 1. 循环依赖问题
**问题**：Service层相互依赖导致DI注册失败
**解决方案**：
- 引入接口抽象，打破直接依赖
- 重构服务分层，明确依赖方向
- 使用工厂模式解决复杂依赖

### 2. 并发数据一致性
**问题**：高并发下的点赞数、阅读量统计
**解决方案**：
```csharp
// Redis原子操作 + 最终一致性
var transaction = _redis.CreateTransaction();
transaction.StringIncrementAsync(likeCountKey);
transaction.SortedSetIncrementAsync(hotPostsKey, postId, 10);
await transaction.ExecuteAsync();
```

### 3. 内存泄漏优化
**问题**：EF Core上下文和Redis连接未正确释放
**解决方案**：
- 统一使用依赖注入管理生命周期
- 实现IDisposable和using语句
- 数据库连接池配置优化

### 4. 深度分页性能
**问题**：传统OFFSET分页在大数据量时性能急剧下降
**解决方案**：
- 游标分页替代OFFSET分页
- 利用雪花ID的有序特性简化游标逻辑
- 针对不同场景选择合适的分页策略
