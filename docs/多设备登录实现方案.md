## 背景与需求

### 问题场景

在现代应用中，用户经常需要在多个设备上同时登录：
- 办公室电脑
- 家里电脑
- 手机
- 平板

传统的单Token方案存在严重问题：

```csharp
// ❌ 传统方案的问题
public class AppUser {
    public string RefreshToken { get; set; }  // 只能存一个！
    public DateTimeOffset? RefreshTokenExpiryTime { get; set; }
}

// 设备A登录 → 生成 Token1
// 设备B登录 → 生成 Token2，覆盖 Token1
// 设备A的Token1失效 ❌
```

**问题**：
- ❌ 后登录的设备会踢掉先登录的设备
- ❌ 用户体验差
- ❌ 无法管理多个设备
- ❌ 无法精确控制单个设备

### 需求分析

1. **多设备支持**：允许用户同时在多个设备上登录
2. **设备管理**：查看所有登录设备，显示设备详情
3. **精确控制**：能够踢出指定设备
4. **安全登出**：登出只影响当前设备
5. **立即生效**：踢出设备后立即失效
6. **会话持久化**：设备信息持久化存储

### 1. JWT结构设计

#### AccessToken Claims

```csharp
public string GenerateToken(AppUser user, IList<string> roles, string? sessionId = null)
{
    var jti = Guid.NewGuid().ToString("N");
    var claims = new List<Claim>
    {
        new(JwtRegisteredClaimNames.Sub, user.Id),
        new(JwtRegisteredClaimNames.Email, user.Email ?? ""),
        new(JwtRegisteredClaimNames.Jti, jti),
        new(ClaimTypes.NameIdentifier, user.Id),
        new(JwtClaimTypes.SessionId, sessionId), // ✨ 关键：SessionID
    };
    
    claims.AddRange(roles.Select(role => new Claim(JwtClaimTypes.Role, role)));
    
    // ... 生成JWT
}
```

#### RefreshToken（JWT格式）

```csharp
public RefreshToken GenerateRefreshTokenJwt(string userId, string sessionId)
{
    var jti = Guid.NewGuid().ToString("N");
    var claims = new List<Claim>
    {
        new(JwtRegisteredClaimNames.Sub, userId),
        new(JwtRegisteredClaimNames.Jti, jti),
        new(ClaimTypes.NameIdentifier, userId),
        new(JwtClaimTypes.SessionId, sessionId),        // ✨ SessionID
        new(JwtClaimTypes.TokenType, "refresh")         // ✨ Token类型
    };
    
    // ... 生成JWT
}
```

**设计亮点**：
- ✅ AccessToken和RefreshToken都包含SessionID
- ✅ RefreshToken也是JWT，可以验证签名
- ✅ 通过SessionID关联同一会话的所有Token

### 2. 会话管理

#### Redis存储结构

```
# 用户的活跃会话集合
user:sessions:{userId} → Set<sessionId>

# 单个会话详情
user:session:{userId}:{sessionId} → {
    "sessionId": "abc123",
    "refreshToken": "eyJhbG...",
    "deviceInfo": "UA:Chrome/Windows|IP:192.168.1.1",
    "loginTime": "2025-01-01T00:00:00Z",
    "lastActiveTime": "2025-01-01T12:00:00Z",
    "userId": "user123"
}

# Token黑名单（立即失效）
token:blacklist:{jti} → expiration
```

#### 会话服务实现

```csharp
public class SessionService : ISessionService
{
    // 创建会话
    public async Task<string> CreateSessionAsync(
        string userId, 
        string refreshToken, 
        string? deviceInfo = null, 
        TimeSpan? sessionTtl = null, 
        string? sessionId = null)
    {
        // 使用预生成的sessionId或生成新的
        sessionId = sessionId ?? Guid.NewGuid().ToString("N");
        
        var sessionInfo = new UserSessionInfo
        {
            SessionId = sessionId,
            RefreshToken = refreshToken,
            DeviceInfo = deviceInfo,
            LoginTime = now,
            LastActiveTime = now,
            UserId = userId
        };
        
        // 存储到Redis
        await _database.StringSetAsync(sessionKey, json, ttl);
        await _database.SetAddAsync(activeSessionsKey, sessionId);
        
        return sessionId;
    }
    
    // 验证会话
    public async Task<UserSessionInfo?> GetSessionAsync(string userId, string sessionId)
    {
        var sessionKey = RedisKeys.Auth.GetUserSessionKey(userId, sessionId);
        var json = await _database.StringGetAsync(sessionKey);
        
        if (json.IsNullOrEmpty)
            return null;
            
        return JsonSerializer.Deserialize<UserSessionInfo>(json!);
    }
    
    // 撤销会话
    public async Task RevokeSessionAsync(string userId, string sessionId)
    {
        var transaction = _database.CreateTransaction();
        
        // 删除会话信息
        transaction.KeyDeleteAsync(sessionKey);
        
        // 从活跃会话集合中移除
        transaction.SetRemoveAsync(activeSessionsKey, sessionId);
        
        await transaction.ExecuteAsync();
    }
}
```

### 3. 登录流程

```csharp
public async Task<LoginResponseDto> LoginAsync(LoginRequestDto loginRequest)
{
    // 1. 验证用户凭据
    var user = await userManager.FindByEmailAsync(loginRequest.Email);
    var isPasswordValid = await userManager.CheckPasswordAsync(user, loginRequest.Password);
    
    // 2. 预生成SessionID
    var sessionId = Guid.NewGuid().ToString("N");
    
    // 3. 生成Token（包含SessionID）
    var roles = await userManager.GetRolesAsync(user);
    var token = jwtService.GenerateToken(user, roles, sessionId);
    var refreshToken = jwtService.GenerateRefreshTokenJwt(user.Id, sessionId);
    
    // 4. 获取设备信息
    var userAgent = httpContext.Request.Headers["User-Agent"].ToString();
    var ipAddress = httpContext.Connection.RemoteIpAddress?.ToString();
    var deviceInfo = DeviceInfoParser.FormatDeviceInfo(userAgent, ipAddress);
    
    // 5. 创建会话（使用预生成的SessionID）
    await sessionService.CreateSessionAsync(
        user.Id, 
        refreshToken.Token, 
        deviceInfo,
        TimeSpan.FromMinutes(refreshTokenExpiryMinutes),
        sessionId  // ✨ 关键：使用相同的SessionID
    );
    
    // 6. 返回Token
    return new LoginResponseDto
    {
        Token = token,
        RefreshToken = refreshToken.Token,
        RefreshTokenExpires = refreshToken.Expires
    };
}
```

**关键点**：
- ✅ SessionID在Token生成前预先创建
- ✅ AccessToken和RefreshToken使用相同的SessionID
- ✅ 设备信息从HTTP请求中提取
- ✅ 会话存储到Redis，设置过期时间

### 4. Token刷新流程

```csharp
public async Task<RefreshTokenRespDto> RefreshTokenAsync(RefreshTokenReqDto request)
{
    // 1. 验证AccessToken（可以是过期的）
    var principal = jwtService.GetPrincipalFromExpiredToken(request.Token);
    var userId = principal.FindFirstValue(ClaimTypes.NameIdentifier);
    
    // 2. 从RefreshToken中提取SessionID
    var sessionId = jwtService.GetSessionIdFromToken(request.RefreshToken);
    
    // 3. 验证会话是否存在
    var sessionInfo = await sessionService.GetSessionAsync(userId, sessionId);
    if (sessionInfo == null || sessionInfo.RefreshToken != request.RefreshToken)
    {
        throw new BusinessException(BusinessError.TokenInvalid);
    }
    
    // 4. 将旧的AccessToken加入黑名单
    var oldJti = jwtService.GetJtiFromToken(request.Token);
    var oldTokenExpiration = jwtService.GetTokenExpiration(request.Token);
    await sessionService.BlacklistTokenAsync(oldJti, remainingTime);
    
    // 5. 生成新Token（使用相同的SessionID）
    var roles = await userManager.GetRolesAsync(user);
    var newToken = jwtService.GenerateToken(user, roles, sessionId);
    var newRefreshToken = jwtService.GenerateRefreshTokenJwt(userId, sessionId);
    
    // 6. 更新会话
    await sessionService.RevokeSessionAsync(userId, sessionId);
    await sessionService.CreateSessionAsync(
        userId,
        newRefreshToken.Token,
        sessionInfo.DeviceInfo,
        TimeSpan.FromMinutes(refreshTokenExpiryMinutes),
        sessionId  // ✨ 保持SessionID不变
    );
    
    return new RefreshTokenRespDto
    {
        Token = newToken,
        RefreshToken = newRefreshToken.Token
    };
}
```

**关键点**：
- ✅ 从RefreshToken提取SessionID（精确定位会话）
- ✅ 旧AccessToken立即加入黑名单
- ✅ 新Token使用相同SessionID（会话连续性）
- ✅ 更新会话信息和过期时间

### 5. 登出流程

```csharp
public async Task LogoutAsync(string userId, string? sessionId = null)
{
    var user = await userManager.FindByIdAsync(userId);
    
    if (!string.IsNullOrEmpty(sessionId))
    {
        // 只撤销当前会话
        await sessionService.RevokeSessionAsync(userId, sessionId);
        
        // 检查是否还有其他活跃会话
        var remainingSessions = await sessionService.GetActiveSessionsAsync(userId);
        if (!remainingSessions.Any())
        {
            // 如果没有其他会话了，清空数据库记录
            user.RefreshToken = null;
            user.RefreshTokenExpiryTime = null;
            await userManager.UpdateAsync(user);
        }
    }
    else
    {
        // 兼容旧逻辑：撤销所有会话
        await sessionService.RevokeAllSessionsAsync(userId);
        user.RefreshToken = null;
        user.RefreshTokenExpiryTime = null;
        await userManager.UpdateAsync(user);
    }
}
```

**关键点**：
- ✅ 只撤销当前会话，不影响其他设备
- ✅ 智能管理数据库记录
- ✅ 向后兼容旧的"全部登出"逻辑

### 6. JWT验证中间件

```csharp
public class JwtBlacklistMiddleware
{
    public async Task InvokeAsync(HttpContext context, 
        ISessionService sessionService, 
        IJwtService jwtService)
    {
        if (context.User.Identity?.IsAuthenticated == true)
        {
            var token = GetTokenFromHeader(context);
            var jti = jwtService.GetJtiFromToken(token);
            var sessionId = jwtService.GetSessionIdFromToken(token);
            var userId = context.User.FindFirstValue(ClaimTypes.NameIdentifier);
            
            // ✨ 双重验证机制
            
            // 1. 检查Token黑名单（立即失效）
            var isBlacklisted = await sessionService.IsTokenBlacklistedAsync(jti);
            if (isBlacklisted)
            {
                context.Response.StatusCode = 401;
                await context.Response.WriteAsync("Token has been revoked");
                return;
            }
            
            // 2. 检查Session是否存在（设备管理）
            if (!string.IsNullOrEmpty(sessionId) && !string.IsNullOrEmpty(userId))
            {
                var sessionInfo = await sessionService.GetSessionAsync(userId, sessionId);
                if (sessionInfo == null)
                {
                    context.Response.StatusCode = 401;
                    await context.Response.WriteAsync("Session has been revoked");
                    return;
                }
                
                // 3. 异步更新会话活跃时间
                _ = Task.Run(() => 
                    sessionService.UpdateSessionActivityAsync(userId, sessionId));
            }
        }
        
        await next(context);
    }
}
```

**双重验证机制**：
1. **黑名单验证**：检查JTI，用于Token刷新时立即失效旧Token
2. **会话验证**：检查SessionID，用于设备管理（踢出设备立即生效）
3. **活跃时间更新**：异步更新，不阻塞请求

### 1. 设备信息解析

```csharp
public static class DeviceInfoParser
{
    // 格式化设备信息
    public static string FormatDeviceInfo(string? userAgent, string? ipAddress)
    {
        return $"UA:{userAgent ?? "Unknown"}|IP:{ipAddress ?? "Unknown"}";
    }
    
    // 解析User-Agent
    public static (string Browser, string OS, string DeviceType, string DeviceName) 
        ParseUserAgent(string userAgent)
    {
        string browser = DetectBrowser(userAgent);
        string os = DetectOS(userAgent);
        string deviceType = DetectDeviceType(userAgent);
        string deviceName = $"{browser} on {os}";
        
        return (browser, os, deviceType, deviceName);
    }
    
    private static string DetectBrowser(string ua)
    {
        if (Regex.IsMatch(ua, @"Edge\/(\d+)", RegexOptions.IgnoreCase))
            return "Edge";
        if (Regex.IsMatch(ua, @"Chrome\/(\d+)", RegexOptions.IgnoreCase))
            return "Chrome";
        if (Regex.IsMatch(ua, @"Firefox\/(\d+)", RegexOptions.IgnoreCase))
            return "Firefox";
        if (Regex.IsMatch(ua, @"Safari\/(\d+)", RegexOptions.IgnoreCase))
            return "Safari";
        return "Unknown Browser";
    }
    
    private static string DetectOS(string ua)
    {
        if (Regex.IsMatch(ua, @"Windows NT 10.0", RegexOptions.IgnoreCase))
            return "Windows 10";
        if (Regex.IsMatch(ua, @"Mac OS X", RegexOptions.IgnoreCase))
            return "macOS";
        if (Regex.IsMatch(ua, @"Android", RegexOptions.IgnoreCase))
            return "Android";
        if (Regex.IsMatch(ua, @"iPhone|iPad", RegexOptions.IgnoreCase))
            return "iOS";
        return "Unknown OS";
    }
}
```

### 2. 设备管理API

```csharp
[HttpGet("active")]
public async Task<ActionResult<ActiveSessionsResponse>> GetActiveSessions()
{
    var userId = User.FindFirstValue(ClaimTypes.NameIdentifier);
    
    // 从当前token中提取sessionId
    var currentToken = Request.Headers.Authorization
        .FirstOrDefault()?["Bearer ".Length..].Trim();
    var currentSessionId = jwtService.GetSessionIdFromToken(currentToken);
    
    var sessionIds = await sessionService.GetActiveSessionsAsync(userId);
    var sessions = new List<SessionDto>();
    
    foreach (var sessionId in sessionIds)
    {
        var sessionInfo = await sessionService.GetSessionAsync(userId, sessionId);
        if (sessionInfo != null)
        {
            // 提取设备信息
            var userAgent = DeviceInfoParser.ExtractUserAgentFromDeviceInfo(
                sessionInfo.DeviceInfo);
            var ipAddress = DeviceInfoParser.ExtractIpFromDeviceInfo(
                sessionInfo.DeviceInfo);
            
            // 解析设备详情
            var (browser, os, deviceType, deviceName) = 
                DeviceInfoParser.ParseUserAgent(userAgent);
            var location = DeviceInfoParser.GetLocationFromIP(ipAddress);
            
            sessions.Add(new SessionDto
            {
                SessionId = sessionInfo.SessionId,
                DeviceName = deviceName,
                DeviceType = deviceType,
                Browser = browser,
                OperatingSystem = os,
                IpAddress = ipAddress,
                Location = location,
                LoginTime = sessionInfo.LoginTime,
                LastActiveTime = sessionInfo.LastActiveTime,
                IsCurrent = sessionId == currentSessionId  // ✨ 精确识别当前设备
            });
        }
    }
    
    return Ok(new ActiveSessionsResponse { Sessions = sessions });
}
```

### 2. 会话安全

```csharp
// 限制同时登录设备数量
public async Task<IEnumerable<string>> LimitUserSessionsAsync(
    string userId, 
    int maxSessions = 5)
{
    var sessions = await GetActiveSessionsWithInfoAsync(userId);
    
    if (sessions.Count <= maxSessions)
        return Enumerable.Empty<string>();
    
    // 按最后活跃时间排序，保留最新的N个
    var sessionsToRevoke = sessions
        .OrderByDescending(s => s.info.LastActiveTime)
        .Skip(maxSessions)
        .Select(s => s.sessionId)
        .ToList();
    
    foreach (var sessionId in sessionsToRevoke)
    {
        await RevokeSessionAsync(userId, sessionId);
    }
    
    return sessionsToRevoke;
}
```

```csharp
    // 设备1登出
    await authService.LogoutAsync(userId, sessionId1);
    
    // 设备1应该无效
    var sessions = await sessionService.GetActiveSessionsAsync(userId);
    Assert.Single(sessions);
    
    // 设备2应该仍然有效
    var device2Session = await sessionService.GetSessionAsync(
        userId, 
        jwtService.GetSessionIdFromToken(device2.Token));
    Assert.NotNull(device2Session);
}
```

## 最佳实践

### 1. 架构设计

✅ **DO**
- 使用SessionID关联同一会话的所有Token
- RefreshToken也使用JWT格式，包含SessionID
- Redis作为主要存储，数据库作为备份
- 双重验证：黑名单 + 会话验证

❌ **DON'T**
- 不要在数据库中存储单一RefreshToken
- 不要只依赖Token过期时间
- 不要忽略设备信息记录
- 不要阻塞主请求更新活跃时间

### 2. 安全考虑

✅ **DO**
- 限制同时登录设备数量
- 记录异常会话行为
- 实施渐进式安全策略
- 定期清理过期会话

❌ **DON'T**
- 不要在日志中记录完整Token
- 不要忽略IP变化等异常
- 不要允许无限设备登录
- 不要在客户端存储敏感信息

### 3. 性能优化

✅ **DO**
- 使用Redis Pipeline批量操作
- 异步更新非关键数据
- 合理设置会话TTL
- 前端缓存设备列表

❌ **DON'T**
- 不要每次请求都查询数据库
- 不要同步更新活跃时间
- 不要过度日志记录
- 不要频繁刷新设备列表

### 4. 用户体验

✅ **DO**
- 清晰显示设备信息
- 提供批量操作
- 实时更新设备状态
- 友好的错误提示

❌ **DON'T**
- 不要隐藏当前设备标识
- 不要让踢出操作难以撤销
- 不要忽略边界情况
- 不要让UI过于复杂

## 总结

### 核心优势

1. **✅ 真正的多设备支持**
   - 每个设备独立会话
   - 互不干扰
   - 精确控制

2. **✅ 安全可靠**
   - 双重验证机制
   - 立即生效
   - 完整的日志记录

3. **✅ 高性能**
   - Redis缓存
   - 异步操作
   - 批量查询

4. **✅ 良好的用户体验**
   - 设备详情展示
   - 直观的管理界面
   - 实时状态更新

### 技术要点

| 技术点 | 实现方式 | 价值 |
|--------|----------|------|
| SessionID | JWT Claim | 会话标识 |
| Redis | 主存储 | 高性能 |
| 双重验证 | 黑名单+会话 | 安全性 |
| 设备信息 | User-Agent解析 | 用户体验 |
| 异步更新 | Task.Run | 性能 |

### 适用场景

✅ **适合**
- SaaS平台
- 企业应用
- 社交媒体
- 电商平台

❌ **不适合**
- 单设备应用
- 简单内部工具
- 无Redis环境
- 极低延迟要求
