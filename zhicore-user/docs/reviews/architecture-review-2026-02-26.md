## 高级问题

### 问题 1：Refresh Token 无服务端存储与吊销机制

- 级别：**高**
- 涉及文档：`01-架构总览与设计决策.md`
- 问题描述：ADR-1 提到"Refresh Token 泄露需主动吊销"，但整个架构文档中没有任何关于 Refresh Token 服务端存储的设计。没有存储就无法实现吊销。缺少 Token 轮换、设备绑定、用户禁用后 Token 失效等关键安全机制。
- 修正建议：在 Redis 中维护 Refresh Token 白名单；实现 Token Rotation；用户被禁用时清除所有 Token

### 问题 2：拉黑操作的分布式锁粒度不足，存在死锁与竞态风险

- 级别：**高**
- 涉及文档：`06-可靠性与一致性设计.md`
- 问题描述：拉黑操作锁键为 `lock:block:{A}:{B}`，但 A 拉黑 B 同时 B 关注 A 时，两个操作持有不同的锁却操作相同的数据行，可能导致统计计数漂移。
- 修正建议：拉黑操作需获取多把锁并规定加锁顺序（按 userId 大小排序），防止死锁

### 问题 3：签到连续天数计算存在跨月边界 Bug 风险

- 级别：**高**
- 涉及文档：`03-数据架构.md`、`06-可靠性与一致性设计.md`
- 问题描述：签到位图按月存储，BITFIELD 只能扫描当月数据。跨月连续签到（如上月28日到本月3日）只能得到3天而非6天。Redis 位图查询层与 PG 统计层的连续天数语义不一致。
- 修正建议：连续签到天数以 PG `consecutive_days` 为权威数据源，Redis 位图仅用于月度日历展示

### 问题 4：Outbox 表缺少重试次数字段和并发调度保护

- 级别：**高**
- 涉及文档：`03-数据架构.md`、`04-事件驱动架构.md`
- 问题描述：`outbox_events` 表缺少 `retry_count`、`next_retry_at` 字段；缺少 `DEAD` 状态；多实例部署时多个 Dispatcher 同时扫描会导致重复投递。
- 修正建议：补充重试相关字段和 DEAD 状态；使用 `SELECT ... FOR UPDATE SKIP LOCKED` 避免重复投递

### 问题 5：API 鉴权设计存在越权风险（IDOR）

- 级别：**高**
- 涉及文档：`05-API设计.md`
- 问题描述：关注、拉黑、签到等接口路径包含 `{userId}`，但文档没有说明如何校验 JWT 中的用户身份与路径中的 userId 是否一致。用户 A 可通过修改 URL 操作用户 B 的数据。
- 修正建议：写操作接口必须校验 `JWT.userId == path.userId`；或去掉路径中的 userId，从 JWT 上下文获取

## 中级问题

### 问题 6：Cache-Aside 缓存不一致窗口未说明

- 级别：**中**
- 涉及文档：`03-数据架构.md`
- 问题描述：删除缓存失败时残留旧数据最长 600s；并发读写场景可能回填旧数据。
- 修正建议：缓存删除失败时通过延迟双删或 MQ 异步重试兜底

### 问题 7：分布式锁降级方案并发安全隐患

- 级别：**中**
- 涉及文档：`06-可靠性与一致性设计.md`
- 问题描述：Redis 不可用时降级为"数据库唯一约束兜底"，但唯一约束无法保护统计计数的原子更新。
- 修正建议：降级方案中统计计数使用 `UPDATE ... SET count = count + 1` 确保原子性

### 问题 8：分页查询缺少复合索引

- 级别：**中**
- 涉及文档：`03-数据架构.md`、`05-API设计.md`
- 问题描述：API 文档提到利用 `(follower_id, created_at)` 复合索引，但数据架构中只有单列索引。
- 修正建议：将单列索引改为复合索引 `(follower_id, created_at DESC)`

### 问题 9：缺少密码修改、禁用、拉黑等关键事件

- 级别：**中**
- 涉及文档：`02-领域模型与聚合设计.md`、`04-事件驱动架构.md`
- 问题描述：`changePassword()` 和 `activate()/deactivate()` 没有触发领域事件，缺少 `UserBlockedEvent`。
- 修正建议：增加 `UserPasswordChangedEvent`、`UserActivatedEvent`/`UserDeactivatedEvent`、`UserBlockedEvent`

### 问题 10：Feign 内部接口缺少安全隔离

- 级别：**中**
- 涉及文档：`05-API设计.md`
- 问题描述：内部接口与外部接口共用端口，外部用户可绕过网关直接访问批量查询接口。
- 修正建议：内部接口使用独立 URL 前缀（如 `/internal/v1/...`），网关拒绝转发

### 问题 11：聚合边界划分模糊

- 级别：**中**
- 涉及文档：`02-领域模型与聚合设计.md`
- 问题描述：UserFollow、UserBlock、UserCheckIn 画在 User 聚合内部，但实际是独立 Repository，应为独立聚合。
- 修正建议：明确为独立聚合，修正领域概览图

### 问题 12：签到双写顺序存在数据丢失风险

- 级别：**中**
- 涉及文档：`06-可靠性与一致性设计.md`
- 问题描述：PG 写入成功后 Redis 写入前应用崩溃，月度签到记录查询会显示未签到。
- 修正建议：查询时优先查 Redis，miss 时回查 PG 并重建位图

### 问题 13：用户注册缺少幂等性设计

- 级别：**中**
- 涉及文档：`05-API设计.md`、`06-可靠性与一致性设计.md`
- 问题描述：网络超时重试可能重复提交注册，唯一约束异常返回 500 而非友好提示。
- 修正建议：注册接口增加幂等键，重复请求返回已注册信息或 409
