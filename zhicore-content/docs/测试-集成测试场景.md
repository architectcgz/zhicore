# zhicore-content 集成测试场景清单（覆盖说明）

## 1. 背景与目标

本文档用于指导 `zhicore-content`（文章服务）集成测试的设计与实施，目标是：

1. 覆盖核心业务链路：文章草稿/发布/定时发布、列表分页、标签、点赞收藏、Outbox 管理端能力。
2. 覆盖关键非功能语义：幂等、并发冲突（乐观锁）、缓存一致性（三态缓存与失效策略）、可观测错误码。
3. 以“真实基础设施”验证为主：数据库、Redis、MongoDB 通过 Testcontainers 提供。

## 2. 测试范围

### 2.1 In Scope（必须覆盖）

- HTTP API（Controller + Facade + Application Service）端到端集成测试：
  - `PostController`：创建/更新/发布/撤销发布/定时发布/取消定时/删除/恢复、详情/列表、草稿、标签。
  - `TagController`：标签详情、列表、搜索、热门、按标签查询文章。
  - `PostLikeController` / `PostFavoriteController`：点赞/取消点赞/状态/批量状态/计数，收藏同理。
  - `OutboxAdminController`：失败列表、手动重试、429 限频与审计。
- 基础设施一致性：
  - PostgreSQL：核心表结构与关键 SQL（含 MyBatis 映射）。
  - Redis：缓存三态（MISS/NULL/HIT）与 `SCAN` 失效策略。
  - MongoDB：文章内容存储（内容查询与同步）。
- 并发语义：
  - 乐观锁冲突分流：不存在 -> 404；并发冲突 -> 409 + `retry_suggested=true`。
  - 标签差量更新并发冲突：409 + `CONCURRENT_TAG_UPDATE`。
- 定时发布：
  - 未到点不发布、到点幂等发布、扫描入队闸门（CAS + cooldown）、补扫重置闸门。

### 2.2 Out of Scope（本轮不强制，但建议后续补齐）

- RocketMQ/外部消息中间件的真实投递链路（本模块在测试中通常 Mock 掉 MQ 生产者）。
- Nacos/配置中心与注册发现联调（测试 profile 关闭）。
- 与其他微服务（user/upload/id-generator）的真实网络联调（测试使用本地桩或 Mock）。

## 3. 测试环境与约束

### 3.1 基础设施（Testcontainers）

- PostgreSQL：元数据存储（文章、标签、Outbox、定时发布事件等）。
- MongoDB：文章内容存储。
- Redis：缓存与分布式锁。

说明：
- 集成测试基座为 `com.zhicore.content.infrastructure.IntegrationTestBase`。
- 测试环境对 ID 服务采用本地桩（避免依赖外部 `zhicore-id-generator`）。
- 用户身份通过请求头 `X-User-Id`（`UserContextFilter`）注入，无需真实鉴权。

### 3.2 断言风格

- HTTP 状态码 + `ApiResponse.code/message/data` 一并断言。
- 对“需要返回 409/429/503 等 HTTP 状态”的场景，优先断言 HTTP status。
- 对分页，断言“无重复/无遗漏/有序”优先于仅断言条数。

## 4. 场景矩阵（按模块）

> 说明：每个场景建议至少包含：正向 1 条 + 异常 1 条 + 可观测断言（状态码/错误码/缓存或 DB 状态）。

### 4.1 文章（Post）

#### P-01 创建文章草稿
- API：`POST /api/v1/posts`
- 覆盖点：
  - 传入合法标题等字段，返回 `postId`。
  - DB 中 `posts` 存在记录，初始状态符合约定（如 DRAFT）。
- 异常：
  - 缺少必填字段/字段越界 -> 400。

#### P-02 更新文章（元数据/内容）
- API：`PUT /api/v1/posts/{postId}`
- 覆盖点：
  - 更新标题/摘要/封面等字段成功。
  - 触发缓存失效（详情 key 必须被删除）。
- 异常：
  - 非作者更新 -> 403。
  - 文章不存在 -> 404。

#### P-03 发布/撤销发布
- API：`POST /api/v1/posts/{postId}/publish`、`POST /api/v1/posts/{postId}/unpublish`
- 覆盖点：
  - 发布后详情 `publishedAt` 非空；列表可查询到。
  - 撤销后 `publishedAt` 变回空；列表不可查询到。
- 异常：
  - 未发布文章点赞/收藏应被拒绝（由点赞收藏模块场景覆盖）。

#### P-04 定时发布/取消定时
- API：`POST /api/v1/posts/{postId}/schedule`、`DELETE /api/v1/posts/{postId}/schedule`
- 覆盖点：
  - 创建定时发布事件（`scheduled_publish_event`）状态正确。
  - 取消后事件状态/字段更新符合约定。
- 异常：
  - `scheduledAt` 非法（过去时间等）-> 400。

#### P-05 删除/恢复
- API：`DELETE /api/v1/posts/{postId}`、`POST /api/v1/posts/{postId}/restore`
- 覆盖点：
  - 软删除后详情不可访问（或按接口约定返回）。
  - 恢复后重新可访问。

#### P-06 详情与内容
- API：`GET /api/v1/posts/{postId}`、`GET /api/v1/posts/{postId}/content`
- 覆盖点：
  - `publishedAt` 映射正确：未发布为 `null`，发布后为实际发布时间。
  - Mongo 内容查询与返回结构正确。

### 4.2 列表分页与参数安全（List/Pagination）

#### L-01 page/cursor 参数互斥与非法参数
- API：`GET /api/v1/posts`
- 覆盖点：
  - `sort/status/page/size` 合法组合返回 200。
  - 非法 `sort/status/page/size` 返回 400，不应 500。

#### L-02 复合游标分页（published_at,id）
- API：`GET /api/v1/posts?cursor=...`
- 覆盖点：
  - `ORDER BY published_at DESC, id DESC` 的稳定有序。
  - 翻页无重复、无遗漏。

#### L-03 混合分页（Hybrid）
- API：`GET /api/v1/posts/hybrid`
- 覆盖点：
  - page 与 cursor 在边界情况下行为一致（不重叠、不跳页）。

### 4.3 草稿（Draft）

#### D-01 保存/读取/删除草稿
- API：`POST /api/v1/posts/{postId}/draft`、`GET /api/v1/posts/{postId}/draft`、`DELETE /api/v1/posts/{postId}/draft`
- 覆盖点：
  - 保存后可读取；删除后读取应返回空或 404（按实现约定）。

#### D-02 草稿列表
- API：`GET /api/v1/posts/drafts`
- 覆盖点：
  - 同一用户多篇草稿排序与字段正确；不同用户隔离。

### 4.4 标签（Tag）

#### T-01 标签创建与查询
- API：`POST /api/v1/posts/{postId}/tags`、`GET /api/v1/tags/{slug}`
- 覆盖点：
  - 不存在标签自动创建；slug 规范化正确。
  - 批量创建“全成全败”：任一失败整批回滚，并返回失败详情（`ValidationException.fieldErrors`）。

#### T-02 标签差量更新与并发冲突
- API：`POST /api/v1/posts/{postId}/tags`、`DELETE /api/v1/posts/{postId}/tags/{slug}`
- 覆盖点：
  - attach/detach 采用差量算法（只变更差异集合）。
  - 并发冲突返回 409 + `CONCURRENT_TAG_UPDATE`。

#### T-03 TagController 列表/搜索/热门/按标签查文章
- API：`GET /api/v1/tags`、`GET /api/v1/tags/search`、`GET /api/v1/tags/hot`、`GET /api/v1/tags/{slug}/posts`
- 覆盖点：
  - 分页、搜索关键词、热门标签输出结构正确。

### 4.5 点赞（Like）与收藏（Favorite）

#### LF-01 点赞/取消点赞/状态/计数
- API：`POST /api/v1/posts/{postId}/like`、`DELETE /api/v1/posts/{postId}/like`、`GET /api/v1/posts/{postId}/like/status`、`GET /api/v1/posts/{postId}/like/count`
- 覆盖点：
  - 仅对 PUBLISHED 文章允许点赞。
  - 状态查询 Redis MISS -> DB -> 回填 Redis 的路径可验证。

#### LF-02 批量状态查询（Pipeline）
- API：`POST /api/v1/posts/like/batch-status`、收藏同理
- 覆盖点：
  - Redis 命中与 DB 回填分支都可覆盖。

### 4.6 Outbox 管理端（R14）

#### O-01 失败列表查询
- API：`GET /api/v1/admin/outbox/failed`
- 覆盖点：
  - 仅返回 FAILED 状态；支持按 eventType 过滤；分页边界处理。

#### O-02 手动重试 + 429 限频 + 审计
- API：`POST /api/v1/admin/outbox/{eventId}/retry`
- 覆盖点：
  - 第一次重试将状态置为 PENDING，写入审计记录（operator_id/reason）。
  - 10 分钟内重复重试返回 429。

### 4.7 定时发布扫描（R1）

#### S-01 scanAndEnqueue 入队闸门（CAS）
- 入口：`ScheduledPublishScanner.scanAndEnqueue()`
- 覆盖点：
  - 到点事件仅入队一次（affected_rows==1）。
  - 冷却时间内重复扫描不应重复入队。

#### S-02 resetStaleGate 补扫重置
- 入口：`ScheduledPublishScanner.resetStaleGate()`
- 覆盖点：
  - 超过 10 分钟的 last_enqueue_at 被置空，便于下一轮扫描重新入队。

### 4.8 缓存三态与失效（R11/R13）

#### C-01 详情缓存三态（MISS/NULL/HIT）
- 覆盖点：
  - MISS：首次读取无缓存 -> 走 DB。
  - NULL：资源不存在写 NULL 缓存（短 TTL）。
  - HIT：重复读取命中缓存（长 TTL）。

#### C-02 列表缓存键维度与失效策略（SCAN）
- 覆盖点：
  - key 包含 `status/sort/size/page|cursor`，避免不同维度互相污染。
  - 变更后 `deletePattern` 使用 SCAN 失效相关列表 key。
