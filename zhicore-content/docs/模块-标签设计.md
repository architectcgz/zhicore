# 模块设计：标签管理（接手版）

## 1. 模块目标
提供标签查询、标签搜索、热门标签以及文章标签绑定能力，支持前台展示与内容分发场景。

## 2. 核心入口
- `TagController`（标签查询）
- `PostController` 的标签相关接口（文章标签管理）
- 业务类：`TagApplicationService`、`PostApplicationService#replacePostTags`

## 3. 标签查询
- 详情：按 slug 查询
- 列表：分页查询
- 搜索：名称模糊匹配
- 标签下文章：先查关系表 postIds，再批量查 Post，避免 N+1
- 热门标签：读缓存，miss 回源 `tag_stats`，再回填缓存

## 4. 标签写入（文章标签替换）
1. 校验文章归属。
2. 校验标签数量上限。
3. 删除旧关系。
4. `findOrCreateBatch` 获取新标签集合。
5. 批量插入新关系。
6. 发布 `PostTagsUpdatedIntegrationEvent`。

## 5. 缓存策略
- 热门标签走 Redis（带 TTL + 随机抖动）
- 空结果缓存防穿透

## 6. 风险点
- `detachTag` 为“读取剩余标签后全量替换”，并发下可能覆盖他人更新。
- 热门缓存失效使用 `keys(pattern)`，在大规模 key 空间下有风险。

## 7. 排障步骤
1. 查 `tags` 与 `post_tags` 关系是否正确。
2. 查 `tag_stats` 是否更新。
3. 检查热门缓存是否过期或失效失败。
4. 查 `PostTagsUpdated` 事件是否被下游处理。

## 8. 改造建议
- 标签更新改为“增删差量操作 + 版本校验”。
- 缓存失效改 `SCAN`，禁用 `KEYS`。
- 为高频标签增加统计异步聚合，降低写放大。
