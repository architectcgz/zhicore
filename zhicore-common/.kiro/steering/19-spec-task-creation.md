# Spec 任务创建规范

[返回索引](./README-zh.md)

---

## 核心规则

**在创建 spec 的 tasks.md 文件时，必须在文件开头添加"执行规范"部分。**

---

## 任务文件结构要求

每个 `tasks.md` 文件必须包含以下结构：

```markdown
# 实施计划：[功能名称]

## 重要提示：执行规范

**在执行本任务列表中的任何任务之前，必须严格遵循以下规范：**

1. **PowerShell 脚本规范** (`.kiro/steering/development/09-powershell.md`)
   - ⚠️ **创建任何脚本之前必须先征得用户同意**
   - 必须说明脚本用途并等待用户确认
   - 所有脚本必须使用 UTF-8 无 BOM 编码

2. **Docker 使用规范** (`.kiro/steering/development/08-docker.md`)
   - 永远使用 `docker-compose` 配置文件管理容器
   - 禁止直接使用 `docker` 命令操作容器

3. **安全规范** (`.kiro/steering/development/12-security.md`)
   - 禁止在代码或日志中暴露敏感信息
   - 所有敏感配置必须支持环境变量覆盖

4. **其他相关规范**
   - 参考 `.kiro/steering/development/` 目录下的所有规范文档
   - 遇到不确定的情况，优先查阅规范或询问用户

**违反规范的后果：**
- 任务执行可能被中止
- 需要重新执行任务
- 可能导致系统不一致或安全问题

---

## Overview

[任务概述内容]

## Tasks

[具体任务列表]
```

---

## 为什么需要这个规范

### 问题背景

在执行任务时，AI 助手可能会：
- 忘记查阅项目规范
- 为了"效率"而跳过必要的确认步骤
- 假设用户需求而不询问

### 解决方案

在任务文件开头明确列出关键规范，确保：
1. **执行者首先看到规范要求**
2. **减少违反规范的可能性**
3. **提供规范文档的快速引用**

---

## 适用范围

此规范适用于：
- 所有 spec 的 tasks.md 文件
- 在 Task Creation Phase 创建任务文件时
- 更新现有任务文件时（如果缺少规范部分）

---

## 执行检查清单

创建 tasks.md 文件时，确保：

- [ ] 文件开头包含"重要提示：执行规范"部分
- [ ] 列出了 PowerShell 脚本规范（最重要）
- [ ] 列出了 Docker 使用规范
- [ ] 列出了安全规范
- [ ] 提供了规范文档的路径引用
- [ ] 说明了违反规范的后果

## 示例

### 正确示例

```markdown
# 实施计划：用户认证功能

## 重要提示：执行规范

**在执行本任务列表中的任何任务之前，必须严格遵循以下规范：**

1. **PowerShell 脚本规范** (`.kiro/steering/development/09-powershell.md`)
   - ⚠️ **创建任何脚本之前必须先征得用户同意**
   ...

## Overview
...
```

### 错误示例

```markdown
# 实施计划：用户认证功能

## Overview
...

## Tasks
...
```

❌ 缺少"执行规范"部分
